# 工资时钟 UI 改版设计规格

日期：2026-05-06

## 背景

工资时钟是一个 Windows 桌面小窗，用来实时显示工作日工资增长。现有主界面已经具备核心信息：今日已赚、累计工资、当前速度、有效工时进度、状态、设置、日历、最小化和记录入口。

本次只优化 UI 表现和信息层级，不修改工资、工时、节假日、历史记录和开机自启等计算或存储逻辑。

外部参考方向来自实时收入和时间追踪类产品：

- CashClock 类产品强调实时收入大数字、简单设置和低干扰界面。
- Time-Pop、Chronum、WorkClock 类产品强调私有本地数据、轻量仪表盘、清晰状态和历史概览。

最终选择方向：轻量财务仪表盘，但保持当前小窗尺寸。

## 已确认设计方向

采用“B 方案：黑白紧凑仪表盘”。

约束：

- 主窗口保持当前小窗规格，目标尺寸继续约 `430x430`。
- 风格为黑白灰极简，不使用绿色或金币风格作为主视觉。
- 保留现有四个操作入口：设置、日历、隐藏、记录。
- 四个按钮加入细线小图标和短文字，设置按钮作为主要入口更醒目。
- 优先改 `wage_clock/theme.py` 和 `wage_clock/app.py`，必要时少量触及弹窗 UI 文件。

草图参考：

- `.superpowers/brainstorm/ui-redesign/b-compact-mono.html`

## 主窗口布局

主窗口继续使用单屏小窗，不增加复杂导航。

从上到下：

1. 顶部标题条
   - 显示“工资时钟”。
   - 保持轻量、低存在感。

2. 深色主金额区
   - 使用近黑色背景和浅色文字。
   - 左上显示“今日已赚”。
   - 右上显示当前状态，例如“工作中”“午休中”“未到工作时间”。
   - 中间显示今日金额，使用最大字号。
   - 底部显示基线信息，例如“已上班 12 天 · 起始累计 ¥7,900.00”。

3. 两张指标卡
   - 左侧：累计工资。
   - 右侧：当前速度。
   - 使用白色卡片、细边框、轻圆角。

4. 有效工时进度区
   - 标题为“今日有效工时”。
   - 右侧显示 `当前工时 / 总工时`。
   - 下方显示黑色进度条。

5. 底部操作按钮
   - 四列均分。
   - 每个按钮包含细线图标和文字。
   - 设置按钮使用深色底，其余按钮使用白底细边框。

## 弹窗调整

设置弹窗和日历弹窗不需要彻底重做，但应与主窗口风格一致。

设置弹窗：

- 保持表单结构。
- 表单间距更整齐。
- 单选、输入框、开机自启和按钮使用统一黑白灰主题。
- “保存开始”作为主要按钮。

日历弹窗：

- 保留当前月视图、上月/下月、工作日/休息日标识。
- 可以将原红色工作日弱化为更克制的深色标记，避免破坏黑白风格。
- 仍需让“需要上班”和“休息/法定假日”容易区分。

历史记录弹窗：

- 保留表格结构。
- 统一 Treeview 表头、行高、边框和背景色。

## 视觉系统

颜色：

- 背景：`#f8f8f6` 或接近的暖灰白。
- 主文本：`#171717`。
- 次级文本：`#6d6d68`。
- 边框：`#deded8`。
- 深色主区：`#171717`。
- 深色区文字：`#f7f7f2`。

字体：

- 中文：`Microsoft YaHei UI`。
- 数字：`Segoe UI`。
- 不使用随视口缩放的字体。

圆角：

- 主金额区约 10px。
- 指标卡、进度区、按钮约 8-9px。

图标：

- 使用细线风格。
- 图标尺寸约 14px。
- 图标与文字水平排列。
- 若 Tkinter 里直接做 SVG 不方便，可采用 Unicode 符号、轻量图标图片，或用 `tk.Canvas` 绘制简单线性图标。

## 交互行为

本次 UI 改版不改变交互语义。

- 设置：打开设置弹窗。
- 日历：打开当月日历。
- 隐藏：最小化到托盘。
- 记录：打开历史记录。
- 窗口关闭和最小化到托盘行为保持现状。
- 每秒刷新工资、速度、进度和状态的行为保持现状。

## 实现边界

应改动：

- `wage_clock/theme.py`：主题 token、ttk 样式、按钮、输入框、进度条、Treeview 等。
- `wage_clock/app.py`：主窗口布局重组、指标卡、主金额区、按钮图标。
- `wage_clock/settings_dialog.py`：必要的按钮/表单样式适配。
- `wage_clock/calendar_dialog.py`：必要的颜色与边框适配。

不应改动：

- `wage_clock/core.py` 的工资和工时计算逻辑。
- `wage_clock/storage.py` 的配置和历史记录格式。
- `wage_clock/holiday_calendar.py` 的节假日规则。
- 打包产物，除非用户后续明确要求重新打包 exe。

## 验证计划

实现后需要验证：

- 程序可通过 `python salary_clock.py` 启动。
- 未初始化状态仍能弹出设置窗口。
- 设置保存后主窗口能正常刷新。
- 今日已赚、累计工资、当前速度、工时进度和状态都显示完整。
- 430x430 小窗中无文字重叠、按钮截断或布局溢出。
- 设置、日历、隐藏、记录四个入口可点击。
- 日历和历史记录窗口仍能打开并显示内容。

## 决策记录

- 用户选择 B 方向：轻量财务仪表盘。
- 用户要求保持当前小窗尺寸。
- 用户选择黑白极简风格。
- 用户要求四个按钮加入小图案。
